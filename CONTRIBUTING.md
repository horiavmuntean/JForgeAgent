# Contributing to JForgeAgent

First off, thank you for considering contributing to JForgeAgent! It's people like you that make the Java ecosystem such an amazing community.

As an experimental project pushing the boundaries of **Java 26**, **JBang**, and **AI Agents**, we welcome all kinds of contributions—from bug reports to new agent orchestration ideas.

## 🚀 How Can I Contribute?

### 1. Reporting Bugs

- Check the [Issues](https://github.com/gazolla/JForgeAgent/issues) to see if the bug has already been reported.
- If not, open a new issue. Clearly describe the problem, including your environment (OS, Java version) and steps to reproduce.

### 2. Suggesting Enhancements

- We are always looking for ways to make the "Forging" process faster and safer.
- If you have an idea for a new Agent (e.g., a Reviewer Agent or a Tester Agent), open an issue to discuss it!

### 3. Tool Examples

- JForgeAgent thrives on its ability to create tools. If you have interesting prompts or specific Java libraries that JForge should know how to use, share them with us!

### 4. Code Contributions

1. Fork the repository.
2. Create a branch for your feature (`git checkout -b feature/amazing-feature`).
3. Commit your changes (`git commit -m 'feat: add some amazing feature'`).
4. Push to the branch (`git push origin feature/amazing-feature`).
5. Open a Pull Request.

## 🛠 Technical Stack & Requirements

To contribute code, ensure you have:

- **JDK 26+** (We use the latest features!).
- **JBang** installed.
- A **Gemini API Key** for testing agentic flows.

## ⚖️ Code of Conduct

By participating in this project, you agree to maintain a respectful and inclusive environment for everyone.
